入手Vuex

发表于 2018-01-16 | 分类于 Vue | | 阅读次数:

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化，简单来说，Vuex就是一个项目上各种状态的统一管理者，这意味着不再需要单独的去对状态进行监控和改变，

前提 - 引入Vuex

1. npm install vuex --save

2. 新建一个vuex文件夹（这个不是必须的），并在文件夹下新建store.js文件，文件中引入我们的vue和vuex。
import Vue from 'vue';
import Vuex from 'vuex';

3. Vue.use(Vuex);

创建state.js count.vue

1. 在store.js里创建一个常量对象。
const state = {
    count: 1
}

2. 用 export default 封装代码，让外部可以引用
export default new Vuex.Store({
    state
})

3. 在 components 文件夹下新建vue模板 count.vue ，在模板中我们引入我们刚建的 store.js，并在模板中用 {{ $store.state.count }} 输出 count 的值。

<template>
    <div>
        <h2>{{msg}}</h2>
        <hr/>
        <h3>{{$store.state.count}}</h3>
    </div>
</template>
<script>
    import store from '@/vuex/store'
    export default{
        data(){
            return{
                msg:'Hello Vuex',

            }
        },
        store

    }
</script>

4.在store.js文件中加入两个改变state的方法。
const mutations={
    add(state){
        state.count++;
    },
    reduce(state){
        state.count--;
    }
}
注意: 以上皆需导出才可使用,如：
export default new Vuex.Store({
    state,mutations
})

6.在count.vue模板中加入两个按钮，并调用mutations中的方法。
<div>
    <button @click="$store.commit('add')">+</button>
    <button @click="$store.commit('reduce')">-</button>
</div>

state访问状态对象

直接通过store取值

$store.state.count

通过computed的计算属性直接赋值

computed:{
    count(){
            return this.$store.state.count;
    }
}

通过mapState的对象来赋值

import {mapState} from 'vuex';

computed:mapState({
    count:state=>state.count
})

通过mapState的数组来赋值

computed:{
    ...mapState(["count"])
}

过滤器 Getters

对状态进行过滤并计算等操作,就像计算属性一样，getter 的返回值会根据它的依赖被缓存起来，且只有当它的依赖值发生了改变才会被重新计算。

定义过滤器 getters

Getter 接受 state 作为其第一个参数：
const store = new Vuex.Store({
    state: {
        count: 1
    },
    getters: {
        add: state => {
        return state.count += 1;
        }
    }
})

export default new Vuex.Store({
    state,mutations,getters
})

使用过滤器

computed:{
    count(){
        return this.$store.getters.count;
    }
},

利用mapGetters简化 :
import { mapState,mapMutations,mapGetters } from 'vuex';

computed: {
    ...mapGetters(["count"])
}

修改状态

利用Mutations修改状态

在前面两个按钮中，我们已经利用了 $store.commit() 对状态进行了修改：

调用 Mutations 里的方法

//$store.commit( )
<button @click="$store.commit('add')">+</button>
<button @click="$store.commit('reduce')">-</button>

//store.js
const mutations={
    add(state){
        state.count++;
    },
    reduce(state){
        state.count--;
    }
}

对Mutations 里的方法进行传参

//定义
const mutations={
    add(state, n){
        state.count+=n;
    },
    reduce(state){
        state.count--;
    }
}

//使用
<button @click="$store.commit('add',10)">+</button>

简化 Mutations 方法的写法

import { mapState,mapMutations } from 'vuex';
...
methods: {
...mapMutations(["add", "reduce"])
}

//使用 - 即可直接写入
<button @click="reduce">-</button>

actions异步修改状态

定义 actions 方法

const actions ={
    addAction(context){
        context.commit('add',10)
    },
    reduceAction({commit}){
        commit('reduce')
    }
}

调用 actions 里的方法

//context：上下文对象，这里你可以理解称store本身。
//{commit}：直接把commit对象传递过来，可以让方法体逻辑和代码更清晰明了。

<button @click="addAction">+</button>
<button @click="reduceAction">-</button>

import { mapActions } from 'vuex';
...
methods:{
    ...mapActions(['addAction','reduceAction'])
},

module 模块组

Vuex 允许我们将 store 分割成模块（module）。每个模块拥有自己的 state、mutation、action、getter、甚至是嵌套子模块——从上至下进行同样方式的分割：

定义模块组

const moduleA = {
    state: { ... },
    mutations: { ... },
    actions: { ... },
    getters: { ... }
}

const moduleB = {
    state: { ... },
    mutations: { ... },
    actions: { ... }
}

const store = new Vuex.Store({
    modules: {
        a: moduleA,
        b: moduleB
    }
})

store.state.a // -> moduleA 的状态
store.state.b // -> moduleB 的状态

使用模块组

<h3>{{ $store.state.a.count }}</h3>    

or

computed:{
    count(){
        return this.$store.state.a.count;
    }
},
